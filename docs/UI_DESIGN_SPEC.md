# T-Display-S3 AIDA64 · 界面设计系统规范（浅色极简）

> UI Designer 交付物。适用于 T-Display-S3（ST7789 1.9"，170×320 竖屏，LVGL v9）。
> 设计目标：**美观、易用、符合用户习惯**——信息层级清晰、对比度高、关键数据一眼可读。

## 1. 设计原则
- **少即是多**：竖屏 170px 宽，只放最关键的数据，杜绝信息过载（原版"太复杂看不清"的根因）。
- **强对比**：深字配浅底，对比度 ≈15:1，远超 WCAG AA 4.5:1。
- **对称结构**：CPU / GPU 两个镜像区块，用户形成稳定心智模型。
- **状态可见**：连接/断开有明确反馈，数据未到位显示占位符而非"全空"。

## 2. 设计令牌

| 令牌 | 值 | 用途 |
|---|---|---|
| `--bg` | `#F4F5F7` | 屏幕底色（柔白灰） |
| `--surface` | `#FFFFFF` | 卡片 / 弧底 |
| `--border` | `#E5E7EB` | 分隔线 / 描边 |
| `--text` | `#1A1D21` | 主数据（大字号） |
| `--text-2` | `#6B7280` | 标签 / 次级 |
| `--text-3` | `#9CA3AF` | 页脚 / 状态 |
| `--cpu` | `#2563EB` | CPU 强调（蓝） |
| `--gpu` | `#F59E0B` | GPU 强调（琥珀，与 CPU 区分） |
| `--ok` | `#16A34A` | 已连接状态点 |

**字体**：Montserrat 10 / 12 / 14 / 18 / 32（需在 `sdkconfig` 启用，见工程 `sdkconfig.defaults`）。
字重：数据 700、标签 500。

## 3. 布局（170 宽 × 320 高，坐标从上到下）

```
┌─────────────────────────────┐  y0
│ PC Monitor        ● 已连接   │  Header h26（白底，底边 1px 分隔线）
├─────────────────────────────┤  y26
│ CPU                         │  y32 标签（蓝）
│   ◔ 45%      温度  62°C      │  弧 96×96 @ (6,46)，中心大字号 32
│              功率  65 W      │  右列统计 @ x112
│                             │
│ GPU                         │  y168 标签（琥珀）
│   ◔ 30%      温度  55°C      │  弧 @ (6,182)
│              功率  40 W      │  右列统计 @ x112
├─────────────────────────────┤  y292 分隔线
│      WiFi 已连接 · 数据接收中  │  Footer（10，灰）
└─────────────────────────────┘  y320
```

### 组件规范
- **Arc（用量环）**：`size 96`、`range 0–100`、`rotation 135`、`bg_angles 0–270`、`arc_width 10`；
  bg arc `#E5E7EB`，indicator 强调色；中心 `lv_label` 显示百分比（32 号）；隐藏旋钮。
- **Stat（温度 / 功率）**：标签（10，`--text-2`）+ 值（18，`--text`）。
- **StatusDot**：r4，连接绿色 / 断开灰并 500ms 闪烁。
- **占位符**：数据到达前显示 `––`，避免"全空"观感。

## 4. 交互与状态
| 状态 | 表现 |
|---|---|
| 未连接 | 状态点灰闪、文字"连接中"、页脚"等待数据…" |
| 已连接无数据 | 双弧 0%、占位符 `––` |
| 正常 | 双弧实时转动、温度/功率刷新、页脚显示连接信息 |
| 断开重连 | 状态点灰闪，2s 后自动重连（见 `aida64.c`） |

## 5. 可访问性 / 易用性
- 对比度满足 WCAG AA。
- 触摸目标虽非交互界面，但数据字体 ≥10px 保证 IPS 屏小字可读。
- 颜色编码一致（蓝=CPU、琥珀=GPU），降低认知负担。
